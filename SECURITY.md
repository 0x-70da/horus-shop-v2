# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅ Yes     |
| < 1.0   | ❌ No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not create a public issue**.

Instead, report it responsibly using one of the following methods:

* Email: [security@yourdomain.com](mailto:security@yourdomain.com)
* Or open a private security advisory via GitHub

Please include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if any)

---

## Response Process

After receiving your report:

* We will acknowledge receipt within 48 hours.
* We will investigate and confirm the issue.
* A fix will be prepared and released as soon as possible.
* You will be credited (if desired) after disclosure.

---

## Scope

This policy applies to:

* Application backend (API routes)
* Authentication system
* Database layer
* Environment configuration
* Dependencies

---

## Responsible Disclosure

We ask that you:

* Do not exploit the vulnerability beyond what is necessary to demonstrate it.
* Do not publicly disclose the issue until it has been resolved.
* Avoid accessing, modifying, or deleting user data.

We appreciate responsible disclosure and thank you for helping keep this project secure.
